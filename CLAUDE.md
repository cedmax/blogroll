# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## Commands

```bash
make build    # fetch all feeds + astro build → dist/
make render   # skip fetching, re-render from cache.json only (go run main.go -skip-fetch)
make dev      # render from cache + start Astro dev server (localhost:4321)
make clean    # remove dist/, src/data/blogroll.json, node_modules/

npm run build   # same as make build (prebuild runs go run main.go)
npm run dev     # same as make dev
npm run preview # serve dist/ locally

go run main.go                # fetch feeds + write src/data/blogroll.json + copy OPML
go run main.go -skip-fetch    # skip fetching, rebuild JSON from cache only
go run main.go -opml <file>   # use a different OPML file
```

## Architecture

Two-stage build: Go fetches feeds and writes JSON; Astro reads JSON and generates all HTML.

```
main.go      → fetches feeds → writes src/data/blogroll.json + public/itblogs.opml
astro build  → reads src/data/blogroll.json → outputs dist/
```

**Data flow:**
1. `parseOPML(itblogs.opml)` → `[]Feed`
2. `fetchAllFeeds()` — parallel HTTP with conditional GET (ETag/Last-Modified), results written to `cache.json`
3. `buildSiteData()` — assembles `SiteData` (groups, feeds, entries) and writes `src/data/blogroll.json`
4. `copyOPML()` → `public/itblogs.opml` (Astro copies `public/` into `dist/` as-is)
5. Astro reads `blogroll.json` at build time and generates all HTML in `dist/`

**Key types in `main.go`:**
- `Feed` — title, XML URL, HTML URL, description, slug
- `Entry` — blog name/URL/slug, post title/URL, published time
- `CacheEntry` — ETag, Last-Modified, description, entries per feed URL (keyed in `cache.json`)
- `SiteData` / `JSONFeed` / `JSONGroup` / `JSONEntry` — JSON output types written to `blogroll.json`

**Slug generation:** derived from the feed's HTML URL hostname (e.g. `cedmax.net`); SHA1 fallback for collisions or unparseable URLs.

**Blog page entry selection:** entries within the last 60 days; padded with older entries to reach a minimum of 10 posts. Logic lives in `buildSiteData()`.

**Astro source layout:**
```
src/
  data/blogroll.json        ← generated by go run main.go, gitignored
  layouts/Base.astro        ← HTML shell, all shared CSS (is:global), nav, footer
  components/EntryRow.astro ← shared date|title row; labeled prop adds "ultimo post" eyebrow
  pages/
    index.astro             ← homepage (recent entries grouped by day)
    directory/index.astro   ← all blogs sorted by last post date
    sites/[slug].astro      ← one page per feed (getStaticPaths over data.feeds)
  utils/dates.ts            ← fmtShort/fmtLong using Intl.DateTimeFormat('it-IT')
```

**`activeNav` prop** on `Base.astro` drives nav active state (`"home"`, `"lista"`, or `""` for blog pages).

**Shared CSS components in `Base.astro` (`<style is:global>`):**
- `.entry-row` / `.entry-row-date` / `.entry-row-title` — the date|title grid row used on blog pages and directory
- `.entry-row-date--labeled` modifier adds the "ultimo post" eyebrow label (directory only)
- `.entries` — the card container (white background, border, border-radius)

Page-specific CSS lives in each page's scoped `<style>` block.
